    CSS与HTML结合方式

    # CSS与HTML结合方式

    <!-- 1. 行内样式 -->
<p style="color:white; background-color:red">文本文本</p>

<!-- 2. 内嵌样式 -->
<head>
    <style>
        p{
        	color: red;
    }
    </style>
</head>

<!-- 3. 引入外部CSS文件 -->
<!-- link标签 -->
<link rel = "stylesheet" type = "text/css" href = "green.css"></link>
<!-- @import 用的少-->
<style type="text/css">
    @import url("green.css");
</style>

二者区别：
link是HTML提供的标签，不仅可以加载CSS文件，还可以定义RSS、rel连接属性等；
@import是CSS提供的语法规则，只有导入样式表的作用；

link标签引入的CSS文件在页面加载同时一起加载；
@import引入的CSS，则在页面加载完成后被加载；

link标签因为是HTML元素，所以可以通过JS操作DOM，插入新的link改变样式；
@import无法通过基于文档树的DOM改变。

    # 打印页的样式表

    两种定义打印页样式的方式：
1、<link rel="stylesheet" href="styles/style.css" media="print" />
2、@media print {
...
}
media属性值可以是（常用的三个）：
	screen：默认值，平时用浏览器查看页面；
	print：网页打印预览及打印出来的效果；
	all：两种情况都会适用。

    # CSS阻塞渲染

    从浏览器的角度看，CSS属于“阻塞渲染”的资源。「阻塞渲染」仅是指该资源是否会暂停浏览器的首次页面渲染。

CSS是阻塞资源，是因为浏览器需要下载并解析链接的CSS文件，然后再渲染页面。

CSS所有链接的文件都会被下载下来，只是如果有的文件不必立即应用，那浏览器就不会让它影响页面的渲染，等到页面初始加载后再处理，以便让用户感觉页面加载速度更快。
因此，如果浏览器要加载的响应式页面通过不同的媒体查询链接了4个不同的样式表（分别为不同视口的设备应用的四种样式），那它就会下载4个CSS文件，但在渲染页面之前，它只会解析那个针对当前视口大小的样式表。

    # 元素四边顺序

    某些属性，需要为元素的四条边或者四个顶点分别进行设置，

    border-radius：30px 40px 50px 60px;
padding：10px 20px;

    设置的顺序为：从左上角或者顶部开始，按顺时针依次分配，没被分配到的与其对面的数值一致；

    # 元素属性的继承性

    有些属性，在给自己设置的时候，后代也会被设置上该属性。

    p { 
    border: inherit; 
}
/* 强制性继承*/

    可继承的：
关于文字样式的属性。包括：color、 text-开头的、line-开头的、font-开头的。
列表样式list-开头的。

不可继承的：
关于盒子、定位、布局的属性，都不能继承。

    # calc函数--属性值的数学计算

    .thing{
	width：calc（50%-10px）；
	/* 父元素宽度的一半减去10像素。百分数表明乘以父元素宽度。 */
}
括号内使用加、减、乘、除都没问题。

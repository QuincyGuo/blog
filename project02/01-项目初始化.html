    # 项目描述

    此项目为Vue后台项目管理系统；让用户通过可视化页面，实现对商品数据的增删改查的操作（包括商品品牌数据、商品分类属性数据等数据）；

    项目技术栈：页面通过Element-UI组件库搭建；借助ECharts实现页面图表的绘制。

    主要工作内容：数据在页面上的显示逻辑。

    # 新建项目

    基于模板进行二次开发。

    # 模板介绍

    https://github.com/PanJiaChen/vue-admin-template
模板介绍：vue2.0项目模板，包含侧边栏、顶部标题栏、主区域的页面框架。

    # 初始化项目

    下载项目压缩包；
新建文件夹并将压缩包解压到其中；
打开解压的文件夹，运行，输入命令：npm install，安装项目依赖；
安装完毕后，输入命令：npm run dev，查看项目运行结果

    # 项目目录说明

    build 
----index.js webpack配置文件【很少修改这个文件】
mock
----mock数据的文件夹【mockjs实现，模拟一些假的数据】，项目使用的是真实接口
node modules
----项目依赖的模块
public
----icon图标，静态页面；经常放置一些静态资源；webpack进行打包的时候，会原封不动打包到dist文件夹中。
src
----程序员的源代码
----api文件夹：			涉及请求相关的
----assets文件夹：		里面放置一些静态资源（一般共享的），放在aseets文件夹里面静态资源，在webpack打包的时候，会进行编译
----components文件夹：	放置非路由组件或全局组件
----icons文件夹：		一些svg矢量图
----layout文件夹：		页面框架些组件与混入
----router文件夹：		路由相关
----store文件夹：		vuex相关
----style文件夹：		样式相关
----utils文件夹：		其中的request.js是axios二次封装文件
----views文件夹：		路由组件
App.vue：根组件
main.js：入口文件
permission.js：与导航守卫相关
settings：项目配置项文件

    # 项目路由配置

    页面整体分为上、左、下三部分，上部为顶部标题栏，下部为主要内容展示区，左部为侧边栏，通过切换左侧下拉框，实现下部主要内容区域的路由切换。

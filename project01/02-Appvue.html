    # 生命周期

    # mounted ()

    this.$store.dispatch('categoryList');
// 调用vuex-actions-categoryList：向服务器发送”获取商品分类的三级列表数据“的网络请求，如果服务器返回的result正常，则将返回的data保存到state中，以供组件使用。

    将TypeNav组件的获取列表数据请求放到这里是因为，多个路由组件里都会用到TypeNav组件；而每次路由切换时，TypeNav组件都会经历销毁又重新生成的过程，导致又发送一次网络请求操作，占用资源。

    Vuex里设置的Actions、Mutations、State，所有组件都可以通过$store属性访问到，因此此处可以使用为其他组件服务的actions。

    # 其他

    <Footer v-show="$route.meta.show"></Footer>
<!-- 
业务有根据不同页面而显示/隐藏Footer组件的需求。
所以，在router路由器配置部分，给每个路由组件增加了show元信息属性；此处的$route获取当前激活的路由组件的show属性，根据其值的true/false来显示/隐藏Footer组件 -->

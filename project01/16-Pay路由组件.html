    组件介绍

    # 组件介绍

    用户支付页。包括弹出支付二维码弹窗、检测支付状态等。

    # 进入离开组件

    1、由Trade组件，路由跳转进入；参数：orderId 订单id

    2、跳转进入PaySuccess路由组件；参数：无

    # 生命周期

    1、mounted () 调用getPayInfo方法；组件向服务器发送”获取订单支付数据“的网络请求，如果服务器返回的result正常，则将返回的data保存到组件的data-payInfo中，以供组件使用。

    # 数据

    名称 数据类型 所属 数据介绍
    payInfo object data 服务器返回的订单支付数据
    timer 定时器 data 查询支付状态的定时器
    code string data 查询支付状态时，服务器返回的状态码
    orderId string computed trade组件路由跳转携带的参数订单id

    # API

    1、获取订单支付信息

    地址：/api/payment/weixin/createNative/{orderId}  请求方式:get   需要带参数orderId
export const reqGetPayInfo = (orderId) =>
  requests({ url: `/payment/weixin/createNative/${orderId}`, method: "get" });

    2、获取订单支付状态

    地址：/api/payment/weixin/queryPayStatus/{orderId}  请求方式:get   需要带参数orderId
export const reqPayStatus = (orderId) =>
  requests({ url: `/payment/weixin/queryPayStatus/${orderId}`, method: "get" });

    # methods

    1、getPayInfo() 获取订单支付数据

    1、参数：无

2、函数内容：
2.2、调用API-获取订单支付信息；
2.3、请求返回成功，获取服务器传回的订单支付信息，将返回数据保存到组件的data-payInfo中。

    2、open() 立即支付，弹出支付二维码弹窗；使用element-ui组件库及qrcode插件；

    async open () {
  // 1、使用qrcode插件生成支付二维码（参数为支付二维码的url）
  // QRCode.toDataURL：将二维码字符串转变为图片的方法，不写await的话其返回结果是一个promise，写成异步函数，等待其成功后的返回结果就是生成的图片url
  let imgUrl = await QRCode.toDataURL(this.payInfo.codeUrl);
  
  // 2、使用element-ui组件库的messagebox组件，生成一个二维码支付弹窗
  this.$alert(`<img src=${imgUrl}>`, '请微信支付', {
    dangerouslyUseHTMLString: true,
    center: true,                     	// 居中布局
    showCancelButton: true,           	// 显示取消按钮
    cancelButtonText: '支付遇到问题',		// 取消按钮文本
    confirmButtonText: '已完成支付',		 // 确定按钮文本
    showClose: false,                	// 显示弹窗右上角关闭图标
    beforeClose: (type, instance, done) => {
      // type：区分取消|确定按钮，instance：当前组件实例，done：关闭弹出框的方法
      if (type == 'cancel') {
        // 2.1、点击支付有问题：弹出提醒消息，清除定时器并将this.timer置空，关闭弹窗
        alert('请联系管理人员');
        clearInterval(this.timer);
        this.timer = null;
        done();
      }
      else {
        // 2.2、点击支付成功：根据向服务器发送获取订单支付状态而返回的支付状态码来确认是否真的已支付成功；如果服务器返回支付成功，清除定时器并将this.timer置空，关闭弹窗，并跳到下一路由。
        // 这里，为了开发人员方便，注释掉判断条件，使不支付时点击已完成支付也能正常跳转到下一页面
        // if (this.code == 200) {
        clearInterval(this.timer);
        this.timer = null;
        this.$msgbox.close();
        this.$router.push('/paysuccess');
        // }
      }
    }
  });

  // 3、定时向服务器发送查询支付状态的请求；如果服务器返回的数据是已支付成功则关闭弹窗跳转向下一路由
  // 如果还没有定时器则开启一个定时器
  if (!this.timer) {
    this.timer = setInterval(async () => {
      // 定时向服务器发送查询支付状态的请求
      let result = await this.$API.reqPayStatus(this.orderId);
      console.log(result);
      // 如果已支付成功
      if (result.code == 200) {
        // 清除定时器并将this.timer置空
        clearInterval(this.timer);
        this.timer = null;
        // 保存返回的支付状态码
        this.code = result.code;
        // 关闭弹窗，并跳到下一路由
        this.$msgbox.close();
        this.$router.push('/paysuccess');
      }
    }, 2000);
  }
}

    # 功能点：

    1、页面数据动态展示

    组件在mounted阶段，调用getPayInfo方法；组件调用获取订单支付数据的API，向服务器发送请求；服务器返回成功则将返回的data保存到组件的data-payInfo中；页面在展示时，使用相应的属性即可。

    2、立即支付

    用户点击支付按钮，调用open方法；使用qrcode插件生成支付二维码；使用element-ui组件库的messagebox组件，生成一个二维码支付弹窗；让支付二维码在弹窗上展示；定时调用获取订单支付状态查询的API，向服务器发送请求；如果服务器返回的数据是已支付成功，则关闭弹窗，跳转向PaySuccess路由；

    支付期间，用户点击支付有问题按钮，弹出提醒消息，清除定时器并将this.timer置空，关闭弹窗；用户点击已支付按钮，查询服务器返回的支付结果，如果已成功，清除定时器并将this.timer置空，关闭弹窗，跳转向PaySuccess路由；

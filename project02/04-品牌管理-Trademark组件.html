    # el-table el-table-column 表格组件

    <el-table :data=list border>
    <el-table-column label="序号" type="index" width='50px' align="center"> </el-table-column>
    <el-table-column prop="tmName" label="品牌名称" width='width'> </el-table-column>
</el-table>

data() {
    return {
        list: [
        	tmName:'',
        	logoUrl:'',
        ],
}}

// :data属性用于表格数据的收集；其属性值一般是数组；表格是竖向，按照一列一列来完成数据分配的；		border：表格边框

// 序号列：label="序号" type="index"，有这两个字段即可完成序号列的自动填充
// 数据列：prop="tmName"，tmName是data:list中的子属性，有这个字段，就会自动把list数组中每一项的tmName属性，按数组顺序显示到这列中。
// 选择列：每行的第一列是多选控件，handleSelectionChange是选择发生变化时的回调函数。
	<el-table @selection-change="handleSelectionChange">
		<el-table-column type="selection" width="60px"></el-table-column>
	handleSelectionChange (val) {
       //参数val 用户选中的行的数据组成的数组[{行一},{行二}]
       this.selectedImgList = val;
    },
// width='width'，width属性可以分配固定的数值宽度，也可以是width属性值，代表均分；	label是每列的标题

    # 表格单元格内展示非文本数据

    <el-table :data=list border>
    <el-table-column label="品牌logo" width='width'>
        <template slot-scope="{row,$index}">
            <img :src="row.logoUrl" style="width:40px;height:40px;">
            <el-button type="warning" icon="el-icon-edit" size="mini"
                       @click="updateTradeMark(row)">修改</el-button>
        </template>
    </el-table-column>
</el-table>

    这里使用了作用域插槽。作用域插槽的原理一句话就是，子留下插槽，由父组件书写结构外观并传递过去显示。 在这里可见的代码都是父组件的代码；而子组件是单元格，是由element-ui封装好的，在这里见不到。所以，这里书写的img是父组件要传递给子组件单元格里面显示的。

    row应该也是单元格子组件封装好传过来的数据。其内容是data:list中当前行的整行数据。row也可以代表本行数据，作为调用组件内函数的参数

    $index同理，其内容是data:list中当前行的顺序，从0开始计算。

    # el-pagination 分页器

    <el-pagination :total="total"
               :current-page="currentPage"
               :page-size="pageDataCountNow"
               :page-sizes="pageDataCount"
               @current-change=""
               @size-change=""
               layout="prev, pager, next, jumper, ->, sizes, total"
               style="margin-top:20px;text-align:center;">
</el-pagination>

data() {
    return {
        currentPage: 1,   // 分页器当前页
        pageDataCountNow: 3,  // 当前每页要展示的数据个数
        pageDataCount: [3, 5, 10],  // 当前每页要展示的数据个数列表
        total: 0,   // 总共的数据个数
}}

// :total、:current-page、:page-size、:page-sizes 都是分页器需要的数据。分别代表数据总数、目前所在页数、当前每页展示多少条数据、每页展示数据条数的切换数组。
// @current-change是用户切换页码时的触发函数，需要为其书写回调函数。
// @size-change是用户切换每页展示数据条数时的触发函数，需要为其书写回调函数。
// layout是分页器内各个子控件的排列顺序。如上面代码，排列顺序就是上一页、页码、下一页、页码跳转、每页展示条数、页码总数。

    2、上传前的函数：判断上传文件的文件类型和大小
beforeAvatarUpload (file) {
    const isJPG = file.type === 'image/jpeg';
    const isLt2M = file.size / 1024 / 1024 < 2;
    if (!isJPG) {
    	this.$message.error('上传头像图片只能是 JPG 格式!');
    }
    if (!isLt2M) {
    	this.$message.error('上传头像图片大小不能超过 2MB!');
    }
    return isJPG && isLt2M;
},

    3、上传成功后的函数：获取文件上传后，在服务器的路径
handleAvatarSuccess (res, file) {

    # el-upload 上传组件-可上传多个文件

    1、上传组件的定义：
<el-upload action="/dev-api/admin/product/fileUpload"
           list-type="picture-card"
           :on-preview="handlePictureCardPreview"
           :on-remove="handleRemove"
           :on-success="handlerSuccess"
           :file-list="spuImageList">
	<i class="el-icon-plus"></i>
</el-upload>
<el-dialog :visible.sync="dialogVisible">
	<img width="70%" :src="dialogImageUrl">
</el-dialog>

// action：要上传的网站路径		list-type：文件列表的类型（text/picture/picture-card）
// on-preview：	点击文件列表中已上传的文件时的触发函数
// on-remove：	文件列表移除文件时的触发函数
// on-success：	图片上传到服务器成功，执行一次该触发函数
// file-list：	已上传的文件列表的相关数据；是一个对象数组。

// dialogVisible：组件内定义的数据，控制预览图片的对话框的显示与否
// dialogImageUrl：组件内定义的数据，要预览查看的文件地址

// :file-list="spuImageList"，数据的定义要求：
spuImageList首先定义为空；
上传成功后，this.spuImageList = fileList;这时，spuImageList会有上传的服务器返回的文件列表数据，包括name、url、response、size、status等；
传递给服务器时，如果服务器需要的是其他名称的属性（比如imgUrl），则还需要给spuImageList内的每个对象再添加其他属性。

    2、预览图片的回调函数
handlePictureCardPreview (file) {
    this.dialogImageUrl = file.url;
    this.dialogVisible = true;
},

3、删除图片的回调函数
handleRemove (file, fileList) {
    // file参数：代表的是删除的那个图片  filelist：照片墙删除某一张图片以后，剩余的其他的图片
    this.spuImageList = fileList;
},

4、添加图片的回调函数
handlerSuccess (response, file, fileList) {
    // response：上传成功后服务器返回的信息，包括code，data(图片在服务器的路径)，message等    
    // file参数：上传成功的那个图片    
    // filelist：照片墙上传一张图片以后，所有图片的相关数据
    this.spuImageList = fileList;
},

    # 生命周期

    1、mounted ()

    调用getPageList方法；组件向服务器发送”获取品牌列表“的网络请求，如果服务器返回的result正常，则将返回的数据保存到组件的data中，以供后续使用。

    # 数据

    名称 数据类型 所属 数据介绍
    currentPage number data 分页器-当前页
    total number data 分页器-总共的数据个数
    page-size number data 分页器-当前每页展示多少条数据
    page-sizes 数组 data 分页器-每页展示数据条数的切换数组
    list 数组 data 当前页要展示的数据列表
    dialogFormVisible Boolean data 添加/修改品牌的对话框的显示隐藏
    tradeMarkForm 对象 data 增加/修改的品牌信息
    rules 对象 data 品牌信息数据的验证规则
    validatePass 函数 data 品牌信息的校验规则，仅限data内部的数据rules使用

    # API

    1、获取品牌列表数据

    地址：/admin/product/baseTrademark/{page}/{limit}	请求方式：get	参数：page第几页	limit每页数据展示个数
export const reqTradeMarkList = (page, limit) =>
	request({ url: `/admin/product/baseTrademark/${page}/${limit}`, method: "get" });

    2、新增/修改品牌数据

    1、新增
地址：/admin/product/baseTrademark/save		请求方式：post		参数：data{tmName品牌名称，logoUrl品牌logo图片路径}
2、修改
地址：/admin/product/baseTrademark/update		请求方式：put		参数：data{id品牌id，tmName品牌名称，logoUrl品牌logo图片路径}
两个API的区别：新增品牌传给服务器的参数没有id，所以将两个请求合一，通过参数有无id进行区分。

export const reqAddOrUpdateTradeMark = (tradeMark) => {
	if (tradeMark.id) {
    	return request({ url: `/admin/product/baseTrademark/update`, method: "put", data: tradeMark });
    } 
	else {
    	return request({ url: `/admin/product/baseTrademark/save`, method: "post", data: tradeMark });
    }
};

    3、删除品牌数据

    地址：/admin/product/baseTrademark/remove/{id} 	请求方式：delete		参数：id品牌id
export const reqDeleteTradeMark = (id) =>
  request({ url: `/admin/product/baseTrademark/remove/${id}`, method: "delete" });

    # methods

    1、getPageList() 获取品牌列表数据

    1、参数：无

2、函数内容：
2.1、获取到当前页码等数据，调用API-获取品牌列表；等待服务器返回结果；如果结果成功，将服务器返回的品牌列表数据保存到组件data中；
2.2、失败则提示失败原因；

    2、addTradeMark() 新增品牌

    1、参数：无

2、函数内容：
2.1、显示新增/修改品牌对话框；
2.2、将收集品牌信息的data:tradeMarkForm置为空，等待用户输入。

    3、updateTradeMark(row) 修改品牌

    1、参数：row 待修改的品牌信息数据

2、函数内容：
2.1、显示新增/修改品牌对话框；
2.2、将 row 赋给 data:tradeMarkForm，等待用户修改。

    4、addOrUpdateTradeMark() 确定新增/修改

    1、参数：无

2、函数内容：
2.1、判断表单内的元素是否全部通过验证；
2.2、如果验证通过，则调用API-新增/修改品牌数据，等待服务器返回结果；如果返回结果成功，则弹出消息提示，关闭对话框，并切换页码至合适页码（修改：当前页；新增：最后一页）；
2.3、如果返回结果失败，则提示失败原因；

    5、deleteTradeMark(row) 删除品牌

    1、参数：row 待删除的品牌信息数据

2、函数内容：
2.1、弹出消息框提示是否确定删除；
2.2、确定删除，则调用API-删除品牌数据，等待服务器返回结果；如果返回结果成功，则弹出消息提示，并切换页码至合适页码（判断本页是否还有数据，有则停留本页，没有则切换至上一页）；
2.3、取消删除，则弹出消息提示。

    6、 beforeAvatarUpload (file) 上传到组件前的函数

    1、参数：file 待上传的文件

2、函数内容：
2.1、判断待上传文件的类型和大小；
2.2、不满足条件，则弹出消息提示，并return false。
2.3、满足条件，不做干涉，upLoad组件会自动将文件上传到设置好的路径。

    7、handleAvatarSuccess (res, file) 上传到组件成功的函数

    1、参数：res, file 上传成功后，服务器返回给前端的数据；实际上要看后端具体是如何定义

2、函数内容：
2.1、获取服务器返回的数据，并将其中的上传成功后的文件的路径，赋值给data:tradeMarkForm，以在页面上显示。

    8、handlerCurrentChange (page) 切换页码

    1、参数：page 切换后新的页码

2、函数内容：
2.1、将切换页码后新的页码page赋值给data:currentPage；
2.2、调用methods:getPageList()，获取本页的品牌列表数据

    9、handleSizeChange(count) 切换每页列表数据的个数

    1、参数：count 切换后新的每页数据的个数

2、函数内容：
2.1、将切换页码后新的每页数据的个数count赋值给data:pageDataCountNow；
2.2、调用methods:getPageList()，获取本页的品牌列表数据

    # 功能点

    1、页面数据动态展示

    组件在mounted阶段，获取到初始的当前页码等数据；之后调用获取品牌列表数据的API，向服务器发送请求；服务器返回成功则将返回的数据保存到组件data中；页面在展示时，使用相应的属性即可。

    用户使用阶段，切换页码、切换每页数据个数等操作，组件获取到当前页码等数据；...（与上面一致）。

    2、新增品牌数据

    用户点击新增按钮，弹出新增品牌的对话框；

    用户在对话框进行输入数据、上传文件等操作，用户操作的同时组件对数据进行验证，以确保新增数据的合法性；

    用户点击确定新增，组件判断表单所有的验证是否全部通过；全部通过则调用新增品牌数据的API，向服务器发送请求；服务器返回成功则弹出消息框提示，关闭对话框并切换至合适的页码。

    3、修改品牌数据

    用户点击修改按钮，弹出修改品牌的对话框；并将待修改的数据显示到表单中；

    用户在对话框上的操作和新增数据一致，操作的同时进行验证；

    用户点击确定修改，验证全部通过后，调用修改品牌数据的API，向服务器发送请求；服务器返回成功则弹出消息框提示，关闭对话框并切换至合适的页码。

    4、删除品牌数据

    用户点击删除按钮，弹出是否确定删除的消息框；确定删除后，则调用删除品牌数据的API，向服务器发送请求；服务器返回成功则弹出消息框提示，切换至合适的页码。

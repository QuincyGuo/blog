    相对定位 position: relative

    # 相对定位 position: relative

    让元素相对于自己原来的位置，进行位置移动。

    position: relative;
left: xx;
top: xx;
<!--
1. 相对定位不脱标:
相对定位的真实位置还在原地，只是显示的时候飘出去其他地方，原地会留下空白。

2. 用途:
微调元素；
做绝对定位的参考；子绝父相。
-->

    # 位置计算

    那条边距离本身对应边的距离。正数向原盒子内偏移，负数向原盒子外偏移。比如left就是看新left边距离原来left边移动距离。

    # 绝对定位 position: absolute

    让元素相对于父容器的四边位置移动。

    position: absolute;
left: xx;
top/bottom: xx;
<!-- 
1. 绝对定位脱标
1.1 标准文档流的性质，绝对定位以后就不遵守了。
	标签不区分行内，块级元素，
	不需要display：block就可以设置宽高了，
	自己的位置根据设置显示，原来的位置会被后面的元素占据。
1.2 相当于自己在标准文档流这个默认图层上新创建了一层图层，和标准文档流的唯一联系是相对于父容器的四边的距离。

2. 如果没有参考父容器
则以当前网页的页面四边，而不是body的边框为参考，做位置移动。
-->

    # 位置计算

    那条边距离父容器对应边的距离。正数向内偏移，负数向外偏移。

    如果不写left等属性，值会是auto，元素位置就会根据周边盒子不同，位置也会不同。

    # 子绝父相

    工程上经常会这样做：父亲设置相对定位（但偏移值为0）作参考，然后让儿子绝对定位一定的距离。

    找距离最近的有相对定位的父元素为参考。

    绝对定位的孩子，不管相对父亲的padding，直接从border内侧就开始定位。

    # 固定定位

    无论页面如何滚动，这个盒子始终都显示，且一直在初始位置不变。参考位置是相对于浏览器页面本身。

    position: fixed;
bottom: 100px;
right: 30px;

    # 位置计算

    距离页面四边的距离。正数向内偏移，负数向外偏移。

    # 三种定位位置计算方式一致

    其实相对、绝对、固定，三种定位方式的偏移位置计算方式都是一致的，只是参考位置不同。

    都是那条边相对参考者的对应边移动，正数向内，负数向外。

    # z-index

    z-index: 1;
<!-- 
当有多个元素重叠时，该属性决定哪些元素排在上面；越大越上。
-->

    选择器的使用思路

    # 选择器的使用思路

    公共类

    不要把每一处的所有样式都只用一个选择器写完；比如
.classA {
    text
    color
    padding
    ...
}
而是要多写独立、简单的样式选择器，然后让HTML标签多携带几个这样的样式，大家共同组成目标样式，比如
.red-text{文本颜色}
.blue-backg{背景颜色}
.padding-10{内间距}

    少用ID

    CSS样式尽量用类选择器，id最好用在行为JS上。

    # 层叠性

    当多个选择器，要为同一个元素，设置同一属性的时候，选择的顺序：

    1. 选择器选择上了目标元素，数各自的权重，(id的数量，类/伪类的数量，标签/伪元素的数量），（如同比较数字大小一样，先比较百位的id数量，再比较十位的类数量，再比较个位的标签数量）。如果权重一样，谁在文件里位置靠后听谁的。

    2. 没有选择上，全都是通过继承影响的，谁的继承距离目标元素更近听谁的。如果继承的一样近，比较选择器权重，如果权重再一样重，谁写在后面听谁的。

    3. 以上顺序，低于行内样式，再低于!important。

      <!--行内样式 -->
<p style = "color:red;">text.</p>

<!--!important -->
.red-text {
	color:red !important;
}

    几个注意的点：

    • 交集选择器div.class1在计算权重时，应该给类选择器和标签选择器都加1。
    • 一个标签引用多个类选择器，比如<div class = “sp1 sp2”>，这个计算的顺序和在这一句里sp1，sp2谁在前后没有关系，只和CSS样式表里选择器引用sp1，sp2的权重计算有关。
    • 后代选择器的只要符合后代关系原则就都会选上。

    # 普通选择器

    /* 标签选择器 */
p { 
    color:red; 
}
/* 类选择器 */
.class { 
    color:red; 
}
/* ID选择器 */
#id { 
    color:red; 
}

    # 复合选择器

    # 后代选择器 空格

    .div p .class1{ 
    /* 每次并不一定会从根节点开始算，每次也并不一定都会完全按照节点树一层一层描述，可能中间会跳一两层。但只要符合这种后代关系的，都可以使用。 */
}

    # 直接子元素选择器 >

    .div > p{
    /* 只选择第一层的子元素，也就是直接子元素。 */
}

    # 交集选择器 紧挨

    p.div{
    /* 一般是标签.类，但是.类.类也是可以的。 */
}

    # 并集选择器 逗号

    p,.div,h1{
    /* 带有任意一个这几种属性的标签都会有该属性。 */
}

    # 属性选择器 []

    主要用于匹配标签的className。

    input[type]{
    /* 存在type属性的input元素。*/
}
input[type="text"]{
    /* 存在type属性并且其值要求是text（不能有其他值并列）的input元素。*/
}
a[class~="download"] {
    /* class属性值包含以空格分隔的多个值，其中一个要求是download的a元素。*/
}
a[class|="download"] {
    /* class属性值只有一个，等于download或者以download开头的a元素。*/
}
a[class*="download"] {
    /* class属性值包含download字符并且在任意位置的a元素。*/
}
a[class^="download"] {
    /* class属性值包含download字符并且在开始位置的a元素。*/
}  
a[class$="download"] {
    /* class属性值包含download字符并且在结束位置的a元素。*/
}        

    input[type='email']:invalid {
    color: red;
}
input[type='email']:valid {
    color: black;
}
/* 属性选择器配合属性值的场景，但使用场景有限，因为:后面的值无法自己指定，只能用预设的值。*/

    # 兄弟元素选择器

    # first-child

    E:first-child 	匹配E元素的父元素的第一个子元素E。
E:last-child 	匹配E元素的父元素的最后一个子元素E。
E:nth-child(n)	匹配E元素的父元素的第n个子元素E。比如： .box:nth-child(6)。
E:nth-last-child(-n+3)，但是n可以从0算起，也就是选倒数第三个第二个第一个共三个元素。
注意，实际选择时，盒子的编号是从1开始算起，不是0。

把E元素的兄弟元素按照顺序从1到N排起来，然后如果第n个的类型是E，那么就选中了，如果类型不一致，那么就没选中。
因此，实际会用在列表中、表格中，这样的兄弟元素都一致的情况下。

    # first-of-type

    E:first-of-type 		匹配同类型中的第一个同级兄弟元素E。
E:last-of-type 			匹配同类型中的最后一个同级兄弟元素E。
E:nth-of-type(n) 		匹配同类型中的第n个同级兄弟元素E。
E:nth-last-of-type(n) 	匹配同类型中的倒数第n个同级兄弟元素E。

既然上面这几个选择器带有type，我们可以这样理解：先在同级里找到所有的E类型，然后根据 n 进行匹配。
这个和上面的第一部分的选择原理是不一样的。

    # 普通同胞结合符 ~

    p~p { 
    /* 选择出现在p元素后的兄弟p元素，不需直接相邻。只要满足关系都选上。 */ 
}

    # 相邻同胞结合符 +

    p+p { 
    /* 选择直接出现在p元素后的兄弟p元素。只要满足关系都选上。 */ 
}

    # 伪类选择器 :单冒号

    静态伪类：只能用于超链接的样式

:link		链接点击之前 
:visited	链接点击之后

    动态伪类：所有标签都适用的样式

:hover 		悬停，鼠标放到标签上时
:focus 		某个标签获得焦点时
:active 	鼠标点击标签，但是还没有松手时

    # 超链接 < a > 标签的伪类

    a:link 、a:visited 、a:hover 、a:active

四个必须按照顺序先后来写，否则会失效。LV HA。记忆梗是：LV哈。
a:link、a:visited，要么同时都写，要么同时都不写，不能只写一个。后面两个a:hover 、a:active可以根据是否需要再写。#### 取反 :not

    .a-div:not(.notme){
	/* 给有a-div类的元素中，选择没有notme类的元素 */
}

    # 多个伪类的交集

    情景如下：每行三个，不知道总数多少，要选出最后一行里的元素（不管最后一行多少）

    Item:nth-child(3n+1):nth-last-child(-n+3)~.Item {
}
:nth-child(3n+1)，选的是第(3n+1)个元素，也就是第一列的元素（n从0开始）；
:nth-last-child(-n+3)，选的是所有的倒数三个元素；
两个紧挨，就是交集，选择的就是最后一行的第一个元素。
然后~.Item，选的就是排在他后面的同级元素。就是最后一行的所有元素。

    # 伪元素选择器

    E::before 设置在 元素E 前面（依据对象树的逻辑结构）的内容，配合content属性一起使用。

    E::after 设置在 元素E 后面（依据对象树的逻辑结构）的内容，配合content属性一起使用。

    content属性必须要有，用来添加文字或图片；如果没有要添加的文字，可以设置成空字符串。

    ::before 和::after 是通过 css 模拟出的html标签的效果。通过CSS给特定元素的前后增加新的元素标签，可以显示在文档模拟树里面的那种，默认是行内元素；可以理解为用CSS手段，在文档树上添加了新的span节点。

    span::after{
  content:"，永不止步";
  color:yellowgreen;
}
<span>生命壹号</span>
<!-- 效果是：生命壹号，永不止步-->

    # 选择第一行、第一个字母

    E::first-line {
    /* 选择元素的第一行，是实际页面呈现出的第一行。 */
}
E::first-letter {
    /* 选择元素的第一个字母 */
}

    # 鼠标悬浮父元素时，子元素发生变化

    父元素的悬浮状态下的后代子元素

    .project-link:hover .code {
  color: #ff7f50;
  /*project-link父元素悬浮时，子元素code设置颜色*/
}

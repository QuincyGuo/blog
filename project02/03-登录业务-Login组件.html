    静态组件

    # 静态组件

    # el-form el-form-item 表单组件

    <el-form ref="loginForm" class="login-form" label-position="left"
         :model="loginForm" :rules="loginRules" :inline="true">
	<el-form-item prop="username">...</el-form-item>
	...
</el-form>	
// :model属性用于表单数据的收集；	:rules属性用于表单数据的规则验证；	:inline属性用于设置一行可以放置多个表单元素
// prop属性是在需要表单数据验证时，必须有的属性，与子表单内的v-model字段名称一致。

    # el-input 输入框组件

    <el-input ref="username" name="username" type="text" placeholder="Username"
          v-model="loginForm.username" 
          @blur="inputBlur(row)"				// 输入框失去焦点时触发
          @keyup.native.enter="inputBlur(row)"	// 键盘输入enter时触发
/>

// 作为子标签嵌套在el-form-item里。v-model属性值与收集本表单数据的data保持一致。

    # 表单规则验证

    1、data的数据名loginRules与el-form:rules的属性值一致；

    2、loginRules内的子数据名与el-form-item的prop属性值一致；

    3、validator为验证器，定义具体的验证规则。

    data() {
	// 验证规则的整体定义：
    return {
        loginRules: {
        	username: [{ required: true, trigger: 'blur', validator: validateUsername }],
        	password: [{ required: true, trigger: 'blur', validator: validatePassword }]
    },
// required: true；必须要有输入；
// trigger：校验的触发事件；blur失焦，change文本改变
// validator：具体的数据验证规则
// message：错误时的提示信息
    
    // 验证规则的具体定义：
    // 1、在内部自定义验证规则：value为待验证的数据
    const validatePassword = (rule, value, callback) => {
        if (value.length < 6) {
            // 不符合验证规则，提示错误
            callback(new Error('The password can not be less than 6 digits'))
        } else {
            // 符合，则继续
            callback()
        }
    }
    // 2、引用别处的验证规则：validUsername为引入的验证函数，其返回结果为true符合/false不符合
    const validateUsername = (rule, value, callback) => {
        if (!validUsername(value)) {
            // 不符合验证规则，提示错误
            callback(new Error('Please enter the correct user name'))
        } else {
            // 符合，则继续
            callback()
        }
    }
}}

    # 数据

    名称 数据类型 所属 数据介绍
    loginForm 对象 data 收集表单数据
    loginRules 对象 data 表单数据的验证规则
    validateUsername 函数 data username的校验规则，仅限data内部的数据使用
    validatePassword 函数 data password的校验规则，仅限data内部的数据使用（相当于函数内的变量）

    # methods

    1、handleLogin 登录业务的处理

    1、参数：无

2、函数内容：
2.1、判断表单内的元素是否全部通过验证；
2.2、如果验证通过，则dispatch触发action:user/login，等待服务器返回结果；如果结果成功，则跳转向dashboard路由组件；
2.3、失败则提示失败原因；

    # vuex

    vuex-user：
1、actions
1.1、login
调用API-用户登录；服务器返回请求成功，则调用mutations-SET_TOKEN，并在本地执行SET_TOKEN操作。
1.2、logout
调用API-用户退出登录；服务器返回请求成功，则调用mutations-RESET_STATE，并在本地执行removeToken操作。
 
2、mutations
2.1、SET_TOKEN
将服务器返回的token数据保存到state中。
2.1、RESET_STATE
将state的数据全部置为默认数据。

    # 功能点

    1、用户登录

    用户在登录表单内输入数据，并点击登录按钮，调用handleLogin方法；然后组件将表单数据进行检验；检验通过后，组件触发vuex的action:user/login；vuex调用用户登录的API，向服务器发送请求；服务器返回成功则跳转至dashboard路由组件。

    2、用户退出登录

    用户点击退出登录按钮，调用logout方法；然后组件触发vuex的action:user/logout；vuex调用退出登录的API，向服务器发送请求；服务器返回成功则跳转至Login登录路由组件。

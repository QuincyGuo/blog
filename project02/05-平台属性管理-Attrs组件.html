    # el-select 下拉选择组件

    <el-select v-model="cForm.category1Id"
           @change="handler1"
           placeholder="请选择" :disabled="isCategoryDisabled" >
	<el-option v-for="(c1) in list1" :key="c1.id"
			   :label="c1.name" :value="c1.id" ></el-option>
</el-select>

// v-model属性用于下拉选择表单元素选中的选项数据与组件data的双向绑定；
// @change是选择的选项发生变化时的触发函数。

// <el-option>用于选项的展示：list1是要展示为选项的数据，通过v-for循环展示出来；	
// label是选项在页面上的展示数据；	value是选项要被<el-select>表单收集的数据。

    多个下拉框组合成一个表单的子表单项&&下拉框要给data传递多个属性的组合数据

    // 多个下拉框组合成一个子表单，作为外面大表单的一个表单项，排在一列。
// 按照下面的data，每个属性是一个下拉框，属性里的每个属性值是下拉框的选项；每个下拉框选择时通过v-model交给data的数据，是在el-option的value里，合并出的新数据，原本不在attrInfoList里；下拉框选则某项后，会给attrInfoList中的每项新加一个属性，存储刚提交选中的合并数据attrIdAndValueId。

    # 两种标签交替出现 数据输入input与展示span

    当页面上是input时，输入完毕挪走鼠标，输入框失去焦点时，切换为span数据展示；

    当页面上是span时，点击span，切换为input数据输入。

    <el-card>
      <el-input v-if="test" @blur="change1"></el-input>
      <span v-else @click="change2" style="display:block;">{{8888}}</span>
</el-card>

data(){
	return{
		test:true,
	}
}
methods:{
    change1 () {
        this.test = false;
    },
    change2 () {
        this.test = true;
    },
}

    # el-popconfirm 气泡确认框

    气泡确认框不会出现在页面中央或顶部，而是会紧挨着出现在元素周围。

    <el-popconfirm :title="`确认删除${row.valueName}？`"
               @onConfirm="deleteAttrValue($index)">
	<el-button slot="reference">删除</el-button>
</el-popconfirm>

// :title是确认框里的文本；		@onConfirm是点击确定按钮时的触发函数（官方库里是@confirm，但项目中，此种写法无用）
// slot="reference" 要放在：触发Popconfirm显示的HTML元素里

    # 组件介绍

    平台内的每个三级分类都有自己的多个属性，每个属性又会有多个属性值。比如，三级分类“手机”，就有运行内存，屏幕尺寸等多个属性，而运行内存属性也有多个属性值，比如4GB、8GB。

    本组件实现选中某个三级分类后，对其属性及属性值进行新增、修改、删除等操作。

    # 进入离开组件

    切换页面左侧的侧边栏，实现对路由的切换。

    对子组件CategorySelect绑定了一个自定义事件getCategoryId，向其传递了一个数据isCategoryDisabled。

    # 子组件

    CategorySelect 平台商品三级分类选择组件，也是一个全局组件。

    # 数据

    名称 数据类型 所属 数据介绍
    category1Id string data CategorySelect组件传递来的所选中一级分类的id
    category2Id string data CategorySelect组件传递来的所选中二级分类的id
    category3Id string data CategorySelect组件传递来的所选中三级分类的id
    attrList Array data 当前三级分类的所有属性及其属性值的数据
    isShowTable Boolean data 属性展示/属性编辑的切换
    attrInfo Object data 当前处于编辑状态的属性数据

    # API

    1、获取分类属性及属性值

    路径：/admin/product/attrInfoList/{categort1Id}/{categort2Id}/{categort3Id}	请求方式：get	参数：categort1Id,categort2Id,categort3Id选中的一二三级分类的id
export const reqAttrList = (categort1Id, categort2Id, categort3Id) =>
	request({url: `/admin/product/attrInfoList/${categort1Id}/${categort2Id}/${categort3Id}`,method: "get" });

    2、保存分类属性属性值

    路径：/admin/product/saveAttrInfo		请求方式：post  	参数：data
export const reqAddOrUpdateAttr = (data) =>
	request({ url: `/admin/product/saveAttrInfo`, method: "post", data });

// 参数data：
{
    attrName: "",
    attrValueList: [				// 属性中的属性值，因为一个属性可有多个属性值，所以数据类型是数组
        {
            attrId: 0,				// 属性值所属的属性的id
            valueName:"",			// 属性值
        }
	],
	categoryId: 0,					// 属性所属的三级分类的id
	categoryLevel:3,				
}

// 不论是新增还是修改，data:attrInfo里保存的一直都是目前正在操作的属性及属性值数据，传给服务器的也都是这个处于编辑状态的数据。

    # methods

    1、getCategoryId (cForm) 绑定给CategorySelect组件的自定义事件的回调函数

    1、参数：cForm CategorySelect组件中选中的一二三级商品分类的id

2、函数内容：
2.1、当CategorySelect组件触发自定义事件时，本组件的回调函数被执行，本组件获取到自定义事件执行时时传过来的参数，将其保存到组件data中。
2.2、调用getAttrList()，从服务器获取当前选中的三级分类的所有属性及属性值

    2、getAttrList() 获取当前三级分类的属性及属性值

    1、参数：无

2、函数内容：
2.1、调用API-获取分类属性及属性值，等待服务器返回结果；如果返回结果成功，将返回的数据保存到data:attrList中；
2.2、失败则提示失败原因。

    3、addAttr() 给分类添加新属性

    1、参数：无

2、函数内容：
2.1、将属性显示切换为属性编辑；
2.2、将data:attrInfo置空，等待用户为其添加数据。

    4、updateAttr(row) 修改分类的某个属性

    1、参数：row 待修改的属性的相关数据

2、函数内容：
2.1、将属性显示切换为属性编辑；
2.2、将row深拷贝给data:attrInfo；
2.3、为attrInfo.attrValueList每个属性值都添加一个isEdit属性，便于确定用户要修改的属性值；

    5、addAttrValue() 为属性添加属性值

    1、参数：无

2、函数内容：
2.1、给attrInfo.attrValueList属性值列表，新增一条默认的属性值数据；
2.2、获取到刚才新增的属性值数据的页面DOM元素，将其获得焦点。

    6、deleteAttrValue(index) 为属性删除属性值

    1、参数：index 待删除的属性值在属性值列表里的顺序索引

2、函数内容：
2.1、调用数组splice方法，根据索引进行删除。

    7、spanFocus (row, index) 将属性值展示切换为属性值输入

    1、参数：row 待修改的属性值相关数据；	index 待修改的属性值在属性值列表里的顺序索引

2、函数内容：
2.1、将当前修改的属性值数据里的isEdit属性变为true；页面DOM会由span切换为input；
2.2、获取到正在修改的属性值数据的页面DOM元素，将其获得焦点。

    8、inputBlur (row) 将属性值输入切换为属性值展示

    1、参数：row 待修改的属性值相关数据；	

2、函数内容：
2.1、对用户输入的新的属性值数据进行验证；验证不通过则进行提示；
2.2、验证通过，则将当前修改的属性值数据里的isEdit属性变为false；页面DOM会由span切换为input；

    9、saveAttrValueList () 保存用户对属性属性值的新增/修改/删除操作

    1、参数：无

2、函数内容：
2.1、对data:attrInfo进行处理，去掉前面新加的isEdit属性（方便编写代码，但是服务器不需要该数据）；
2.2、调用API-保存分类属性属性值，等待服务器返回结果；如果返回结果成功，则弹出消息提示，切换回属性展示页面，调用getAttrList方法，从服务器获取更新后的属性数据；
2.3、如果返回结果失败，则提示失败原因；

// 前面所有的对于属性属性值的新增/修改/删除操作，全部都是对data:attrInfo的操作，所以最终将data:attrInfo这个保存最终编辑结果的数据传递给服务器即可；

    # 功能点

    1、属性及属性值数据动态展示

    子组件里选择好三级分类数据，触发自定义事件，将数据传递过来；之后本组件，调用获取分类属性及属性值的API，向服务器发送请求；服务器返回成功则将返回的数据保存到组件data中；页面在展示时，使用相应的属性即可。

    2、属性、属性值的添加/修改/删除

    用户点击添加/修改属性按钮，属性展示区域切换为属性编辑区域；初始化记录用户操作属性的数据data:attrInfo；把用户的每步操作对应的数据变动，都记录在数据data:attrInfo中；（包括新增属性名、新增属性值、修改已有的属性值、删除已有的属性值）。

    其间的数据操作还包括属性值的input输入和span展示的变动，输入文本的检验，变动数据期间的按钮限制等。

    3、保存对属性、属性值的编辑

    用户点击保存，调用保存属性、属性值数据的API，向服务器发送请求；服务器返回成功则弹出消息框提示，切换回属性展示页面。

    # 子组件 CategorySelect

    # 组件介绍

    平台内的商品分类，总共划分为三级。比如，一级分类有电子产品、服装；服装的一级分类下又有男装、女装等二级分类；男装的二级分类下又有男士上衣、男士裤子等等三级分类。

    本组件就是通过三个下拉选择框，让用户一级一级地选择出某个三级分类；把选择好的三层分类的每层分类的id包装成一个对象，传递给需要的组件。

    # 进入/离开组件

    全局组件，在Attrs组件中作为子组件使用；

    参数：父组件Attrs添加的自定义事件getCategoryId（传出本组件内选择的三级分类的id）、及父组件传入的数据isCategoryDisabled

    # 数据

    名称 数据类型 所属 数据介绍
    list1 Array data 平台商品的所有一级分类数据
    list2 Array data 所选择的一级分类下的所有二级分类数据
    list3 Array data 所选择的二级分类下的所有三级分类数据
    cForm Object data 用户选择的一/二/三级分类的id数据
    isCategoryDisabled Boolean props 外部组件控制本组件内三个下拉框的禁用与否的数据

    # 生命周期

    1、mounted ()

    调用getCategoryList1方法；组件向服务器发送”获取一级商品分类“的网络请求，如果服务器返回的result正常，则将返回的数据保存到组件的data中，以供后续使用。

    # API

    1、获取一级商品分类

    路径：/admin/product/getCategory1		请求方式：get	参数：无
export const reqGetCategory1List = () =>
	request({ url: `/admin/product/getCategory1`, method: "get" });

    2、获取二级商品分类

    路径：/admin/product/getCategory2/{categort1Id}	请求方式：get	参数：categort1Id一级分类的id
export const reqGetCategory2List = (category1Id) =>
	request({ url: `/admin/product/getCategory2/${category1Id}`, method: "get" });

    3、获取三级商品分类

    路径：/admin/product/getCategory3/{categort2Id}	请求方式：get	参数：categort2Id二级分类的id
export const reqGetCategory3List = (category2Id) =>
	request({ url: `/admin/product/getCategory3/${category2Id}`, method: "get" });

    # methods

    1、getCategoryList1 获取一级商品分类数据

    1、参数：无

2、函数内容：
2.1、调用API-获取一级商品分类；等待服务器返回结果；如果结果成功，将服务器返回的一级分类列表数据保存到组件data:list1中；
2.2、失败则提示失败原因；

    2、getCategoryList2 / getCategoryList3 获取二/三级商品分类数据

    原理与getCategoryList1一致。调用相应的API，将服务器返回的相应层级的分类列表数据保存到组件data:list2/list3中；

    3、handler1 选中某个一级分类选项

    1、参数：无

2、函数内容：
2.1、将组件data中关于二级、三级分类的数据全部置空；
2.2、调用getCategoryList2，获取二级商品分类数据。

    4、handler2 选中某个二级分类选项

    1、参数：无

2、函数内容：
2.1、将组件data中关于三级分类的数据全部置空；
2.2、调用getCategoryList3，获取三级商品分类数据。

    5、handler3 选中某个三级分类选项

    1、参数：无

2、函数内容：
2.1、触发Attrs组件绑定的自定义事件getCategoryId，传递数据data:cForm给Attrs组件。

    # 功能点

    1、选择一二三级商品分类

    组件在mounted阶段，调用获取一级分类的API，向服务器发送请求；服务器返回成功则将一级分类列表数据保存到data中；

    选中某个一级分类，调用获取相应一级分类下的二级分类的API，向服务器发送请求；服务器返回成功则将二级分类列表数据保存到data中；

    选中某个二级分类，调用获取相应二级分类下的三级分类的API，向服务器发送请求；服务器返回成功则将三级分类列表数据保存到data中；

    选中某个三级分类，触发Attrs组件绑定的自定义事件getCategoryId，将三个选中的分类id传递给Attrs组件。

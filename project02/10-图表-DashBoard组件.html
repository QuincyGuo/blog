    # 静态组件

    # el-card 卡片上画图表

    BottomLeftCard组件做容器组件，里面划分一级结构，将卡片分为顶部标题和下部的图表区。而图表区又分为左右两个图表，每个图表都引入图表子组件进行绘制。同时通过插槽，向图表子组件传递模板内容进行显示。

    <template>
    <el-card>
		// 1、卡片的顶部标题
        <div slot="header">
            <div class="header1">
            	<span>线上热门搜索</span>
            </div>
        </div>
        // 2、卡片的下部图表区
        <div>
            <el-row :gutter="10">
            	// 2.1、左侧图表，引入图表子组件进行绘制。
                <el-col :span="12">
                	<LineChart>
                		// 2.1.1、向LineChart组件内的footer插槽传递模板内容。
                		<template slot="footer">
              				<span>转化率&nbsp;&nbsp; 68%</span>
            			</template>
                	</LineChart>
                </el-col>
                // 2.2、右侧图表
                <el-col :span="12">
                    <LineChart></LineChart>
                </el-col>
            </el-row>
        </div>
    </el-card>
</template>
export default {
  name: 'BottomLeftCard',
  components: {
    LineChart,
  }
}

    LineChart做图表组件。同样在组件内划分为顶部标题区、中部数据区、中部图表区、底部文本区。其中标题区、数据区是正常文本；图表区定义一个container容器，具体的绘制在script里；底部文本区定义一个插槽，由父组件传递模板内容进行显示。

    <template>
    <div>
    	// 1、标题区、数据区
    	<div class="card-header">
    		<span>搜索用户数</span>
    	</div>
        <div class="card-content">12321 &nbsp;&nbsp; 17.6% <img src="@/assets/up.svg"></div>
        // 2、图表区
        <div class="card-chart">
        	<div class="container" ref="chart1"></div>
        </div>
        // 3、文本区
        <div class="card-footer">
        	<slot name="footer"></slot>
        </div>
    </div>
</template>

    # 组件介绍

    此组件功能是展示图表。将页面划分为不同的card卡片，在卡片内又划分不同的区域，比如标题区、图表区、文本区，具体的划分方法见上面的“卡片上画图表”。

    图表的数据来源是静态写死的，图表的绘制方式见下面代码：

    lineChart组件：
<template>
    <div>
        ...
        // 定义画布容器
        <div class="card-chart">
        	<div class="container" ref="chart1"></div>
        </div>
    </div>
</template>

<script>
// 引入echarts 
import echarts from 'echarts';
export default {
	name: 'LineChart',
  	mounted () {
    	// 初始化ECharts实例
    	let lineChart = echarts.init(this.$refs.chart1);
    	lineChart.setOption({
    		// 给图表添加数据
        	series: 
        	[{
          		type: 'line',
          		data: [10, 5, 16, 12, 8, 9, 15, 13],
        	}],
      		...
    	});
	},
}
</script>

    # 划分结构的复用组件 & 子组件作为插槽传递

    下图中的图表部分，是当前页面的子组件，作为插槽内容被传递给另一个子组件。

    DashboardCard父组件：
<template>
	<el-card>
		<CardDetail title="访问量" count="88498">
        	<template slot="chart">
            	<LineChart></LineChart>
            </template>
            <template slot="footer">
                <span>日访问量&nbsp;&nbsp; 1315</span>
            </template>
        </CardDetail>
	</el-card>
</template>
export default {
	name: 'DashboardCard',
    components: {
    	CardDetail, LineChart
  }
}

    CardDetail子组件：
<template>
    <div>
        <div class="card-header">
            <span>{{title}}</span>
        </div>
        <div class="card-content">{{count}}</div>
        <div class="card-chart">
            <slot name="chart"></slot>
        </div>
        <div class="card-footer">
            <slot name="footer"></slot>
        </div>
    </div>
</template>
export default {
    name: 'CardDetail',
    props:
    	['title', 'count'],
}

    使用CardDetail子组件进行页面区域划分，原因是，项目中有许多地方会用到CardDetail组件这样的区域划分，所以将其定义为一个组件，专门用来区域划分。

    CardDetail子组件分为四个区域。分别是：header区、content区、chart区、footer区。同时定义了具名插槽，由父组件为插槽区域传递模板内容。

    DashboardCard父组件使用这个划分结构的子组件，让其将页面上的某部分区域进行划分。然后使用向其传递数据，不仅传递了props数据，更是将LineChart子组件作为插槽内容，传递给CardDetail子组件进行显示。

    静态组件

    # 静态组件

    # 组件介绍

    spu、sku的介绍：

    spu、sku是商品网站中关于商品关系组织的名词。
spu可以看做是类，sku实例对象。

比如一个“小米10”类的spu：
spu名称：小米10；
spu品牌：小米；
spu图片：若干小米10的图片；
手机属性：
	运行内存：4GB、8GB...
	外壳颜色：红色、银色...

一个“小米10”类的实例对象的sku：
spu名称：小米10；
sku名称：小米10 8GB 黑色；
sku图片：若干小米10 8GB 黑色的图片
手机属性：（这里的手机属性，只能从spu里面已定义的属性里选择）
	运行内存：8GB
	外壳颜色：黑色

    本组件功能：

    实现对spu类的新增、修改、为spu新增sku、查看某个spu类下的所有sku等操作。

    # 进入离开组件

    切换页面左侧的侧边栏，实现对路由的切换。

    对子组件SpuForm / 子组件SkuForm各自绑定了一个自定义事件changeScene，以确定子组件是在进行哪一功能，以及结束子组件后本组件要切换至哪个区域进行展示。

    # 子组件

    SpuForm 添加/修改spu类组件。

    SkuForm 为spu添加sku组件。

    HintButton 对el-button的封装，实现自定义组件-按钮。

    # 数据

    名称 数据类型 所属 数据介绍
    category1Id string data CategorySelect组件传递来的所选中一级分类的id
    category2Id string data CategorySelect组件传递来的所选中二级分类的id
    category3Id string data CategorySelect组件传递来的所选中三级分类的id
    spuList Array data 当前三级分类的所有spu相关的数据
    scene number data spu展示/spu编辑的切换
    currentPage number data spu展示列表-当前页
    pageDataCountNow number data spu展示列表-每页展示的数据个数
    total number data spu展示列表-所有的数据个数
    dialogTableVisible Boolean data 展示某个SPU下的所有SKU实例-对话框的显示/隐藏
    spuOfShowAllSku Object data 展示某个SPU下的所有SKU实例-spu数据
    allSku Array data 展示某个SPU下的所有SKU实例-SKU实例数据

    # API

    1、获取SPU列表

    // 路径：/admin/product/{page}/{limit}		请求方式：get	参数：page第几页	limit每页数据展示个数	catrgory3Id所属的三级商品分类id
export const reqSpuList = (page, limit, category3Id) =>
  request({ url: `/admin/product/${page}/${limit}`, method: "get", params: { category3Id } });

    2、删除某条SPU

    // 路径：/admin/product/deleteSpu/{spuId}		请求方式：delete		参数：spuId待删除的spu的id
export const reqDeleteSpu = (spuId) =>
  request({ url: `/admin/product/deleteSpu/${spuId}`, method: "delete" });

    3、获取某条SPU下的所有SKU

    // 路径：/admin/product/findBySpuId/{spuId}		请求方式：get		参数：spuId待获取sku数据的spu的id
export const reqSkuListBySpuId = (spuId) =>
  request({ url: `/admin/product/findBySpuId/${spuId}`, method: "get" });

    # methods

    1、getCategoryId (cForm) 绑定给getSpuList组件的自定义事件的回调函数

    1、参数：cForm 子组件中选中的一二三级商品分类的id

2、函数内容：
2.1、当getSpuList组件触发自定义事件时，本组件的回调函数被执行，本组件获取到自定义事件执行时时传过来的参数，将其保存到组件data中。
2.2、调用getSpuList()，从服务器获取当前选中的三级分类下的所有spu数据

    2、getSpuList() 获取当前三级分类下的spu列表数据

    1、参数：无

2、函数内容：
2.1、准备参数，调用API-获取SPU列表，等待服务器返回结果；如果返回结果成功，将返回的数据保存到data:spuList中；
2.2、失败则提示失败原因。

    3、addSPU() 添加新spu类

    1、参数：无

2、函数内容：
2.1、将spu展示切换为spu编辑；
2.2、调用子组件spuForm的initAddSpuData()，之后由用户在表单中添加spu数据。

    4、updateSPU(row) 修改某个spu类

    1、参数：row 待修改的spu类的相关数据

2、函数内容：
2.1、将spu展示切换为spu编辑；
2.2、调用子组件spuForm的initSpuData(row)，之后由用户在表单中修改spu数据。

    5、changeScene (index, flag) 给SpuForm组件添加的自定义事件的回调函数

    1、参数：index 控制spu展示/spu编辑的区域切换	flag 控制页码跳转

2、函数内容：
2.1、SpuForm组件完成了spu的添加/修改，触发自定义事件，引发本回调函数的执行；
2.2、将spu编辑切换为spu展示；调用getSpuList()，从服务器获取修改后的所有的spu数据，并进行页码的适当切换；

    6、changeScene2 (index) 给SkuForm组件添加的自定义事件的回调函数

    1、参数：index 控制spu展示/sku编辑的区域切换	

2、函数内容：
2.1、SkuForm组件完成了sku的添加，触发自定义事件，引发本回调函数的执行；
2.2、将sku编辑切换为spu展示。

    7、addSKU(row) 为spu类添加sku实例

    1、参数：row 待添加的spu类的相关数据

2、函数内容：
2.1、将spu展示切换为sku编辑；
2.2、调用子组件skuForm的initSkuData(1Id, 2Id, row)，之后由用户在表单中添加sku数据。

    8、showAllSku(row) 展示某个spu类下的所有sku实例

    1、参数：row 待查看的spu类的相关数据

2、函数内容：
2.1、显示查看sku实例的对话框；
2.2、准备参数，调用API-获取某条SPU下的所有SKU，等待服务器返回结果；如果返回结果成功，将返回的数据保存到data:allSku中，以进行数据展示；

    9、deleteSPU (row) 删除spu

    1、参数：row 待删除的属性值相关数据；	

2、函数内容：
2.1、调用API-删除某条SPU，等待服务器返回结果；如果返回结果成功，则进行消息框提示，调用getSpuList()，从服务器获取删除后的所有的spu数据，并进行页码的适当切换；

    10、handlerCurrentChange (page) 切换页码

    1、参数：page 切换后新的页码

2、函数内容：
2.1、将切换页码后新的页码page赋值给data:currentPage；
2.2、调用methods:getSpuList()，获取本页的spu列表数据

    11、handleSizeChange(count) 切换每页列表数据的个数

    1、参数：count 切换后新的每页数据的个数

2、函数内容：
2.1、将切换页码后新的每页数据的个数count赋值给data:pageDataCountNow；
2.2、调用methods:getSpuList()，获取本页的spu列表数据

    # 功能点

    1、spu数据动态展示

    CategorySelect组件里选择好三级分类数据，触发自定义事件，将数据传递过来；本组件调用获取SPU列表数据的API，向服务器发送请求；服务器返回成功则将返回的数据保存到组件data中；页面在展示时，使用相应的属性即可。

    用户使用过程中，切换页码、切换每页数据个数等操作，组件获取到当前页码等数据；调用获取SPU列表数据的API...（与上面一致）。

    2、SPU的添加/修改

    用户点击添加/修改spu按钮，spu展示区域切换为spu编辑区域；调用SpuForm组件的初始化函数，并向其传递相关的spu相关数据；之后在SpuForm组件中完成对spu的添加/修改工作；

    SpuForm组件完成工作后，触发本组件为其绑定的自定义事件，引发本组件相应的回调函数，spu编辑区域切换为spu展示区域，并重新获取spu数据进行展示。

    3、SPU的删除

    用户点击删除按钮，调用删除spu的API，向服务器发送请求；服务器返回成功则弹出消息框提示，重新获取spu数据进行展示。

    4、SKU的添加

    用户点击添加sku按钮，spu展示区域切换为sku编辑区域；调用SkuForm组件的初始化函数，并向其传递相关的spu相关数据；之后在SkuForm组件中完成对sku的添加工作；

    SkuForm组件完成工作后，触发本组件为其绑定的自定义事件，引发本组件相应的回调函数，sku编辑区域切换为spu展示区域，并重新获取spu数据进行展示。

    5、查看SPU下所属的SKU

    用户点击查看SPU下的SKU按钮，显示查看sku实例的对话框；调用获取spu下的sku的API，向服务器发送请求；服务器返回成功则将返回的数据保存到组件data:allSku中，页面在展示时，使用相应的属性即可。

    用户查看完毕，关闭查看sku实例的对话框，data:allSku置空。

    # 子组件 HintButton

    # 组件介绍

    HintButton 对el-button进行二次封装，通过$attrs，将父组件传递给 HintButton 的属性，再传递给子组件el-button，让el-button使用这些属性。

    <template>
  	<a :title="title">
    	<el-button v-bind="$attrs"
               	   v-on="$listeners"
                   style="margin:10px 10px;"></el-button>
  	</a>
</template>
export default {
  name: "HintButton",
  props: ['title'],
}

// 在父组件Spu中，给HintButton组件绑定了type、size等属性及@click事件；
// 而在HintButton组件中，又通过v-bind="$attrs"、v-on="$listeners"，将这些属性事件又绑定给了子组件el-button。
// 于是就实现了：通过HintButton对el-button进行二次封装，完成按钮组件的定义。

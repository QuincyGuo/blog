    标准盒模型

    # 标准盒模型

    盒子中的主要属性：

    width、height:	内容的宽度、高度（不是盒子的宽度、高度）。
padding：		内容和盒子边之间的内边距。
border：			盒子边框。
margin：			盒子边框和盒子外部的外边距。

<!-- 一个空盒子，如果没有设置宽高，内容，仅靠背景图片是撑不起盒子的。 -->

    # box-sizing: content-box

    content-box：默认属性，盒子的浏览器真实占有宽度 = 内容宽度 + 左右内边距 + 左右border；

    缺点：
该属性设置在响应式布局会比较麻烦。
子容器可能会因为width：100%，再加上设置了padding会超出父容器的边界，造成浏览器右侧超伸，其他地方会有空白。(但是固定定位position：fixed加width:100%的元素还是占据浏览器最后实际的宽度，会与超伸的一样长，估计是运算逻辑在后面)
给盒子设置背景图片或背景色时，padding也会被占据；

    # box-sizing: border-box

    border-box；会让本来是设置的内容宽度的width，变成内容宽度 + 左右padding + 左右border。

    这使得处理元素大小的工作变得容易得多，并且通常消除了在布局内容时可能遇到的许多陷阱。然而，在某些情况下，你应谨慎使用这个属性。例如： 你正在编写一个将由其他人使用的共享组件库，如果他们网站的其余部分没有设置此值，他们可能会发现很难使用你的组件库。

并且，该属性不继承。如果要使用，要用下面的代码。
html {
  box-sizing: border-box;
}
*,
*::before,
*::after {
  box-sizing: inherit;
}

    # 块级盒子 block

    display: block;

    盒子单独霸占一行，不能与其他任何元素并列；
盒子能接受宽、高设置。如果不设置宽度，那么宽度将默认变为父亲的100%；
padding 、margin 、 border 会将其他元素从当前盒子周围“推开”。

    # 内联盒子 inline

    display: inline; 主要有：a 、 span、 em 、 strong

    盒子与其他行内元素并排；
不能设置宽、高。默认的宽高，就是文字的宽高。
垂直方向的padding 、margin 、 border会被应用但是不会把其他处于 inline 状态的盒子推开。
水平方向的padding 、margin 、 border会被应用且会把其他处于 inline 状态的盒子推开。

    看做文本：
img 、 表单元素，不严格属于块级、内联任一种。

    # inline-block

    在内联和块之间提供了一个中间状态：

    设置`width` 和`height` 属性会生效。
padding`, `margin`, 以及`border` 会推开其他元素。
但是，它不会跳转到新行。

    # 盒子的真实占有宽高

    盒子的浏览器真实占有宽度 = 内容宽度 + 左右内边距 + 左右border

    盒子的宽度会变化，但是高度一般都是固定的。

    # width: auto

    宽和高的默认值是auto。

    宽度的auto是，父元素留出的空白宽度，减去元素已设置的padding、margin、border

    # min-height

    min-height比height更适用。

    如果设置了高度，随着内容变多，标准的浏览器不会自动扩大盒子高度，它们有可能撑破元素的盒子，这可能是你预期之外的。
不过，如果希望元素至少具有某一特定高度，可以设置min-height。如果内容日后变多，元素的高度会自动按需增长。

这是height与min-height的区别，width和min-width也是类似的。

    # 相对单位

    # em

    font-size：使用是相对于父元素的字体大小；

    其他属性（比如width）：使用是相对于自身的字体大小。

    # rem

    相对于HTML根元素的字体大小。

    # 百分数

    font-size：使用是相对于父元素的字体大小；

    其他属性（比如width）：百分比数 * 父元素宽度。一般不用于上下边距，因为是根据元素宽度计算的。

    # 行高

    line-height：20px；
<!-- 
行高 = 字号 + 上下padding。
行高也可以用纯数字，表示大小是字号的多少倍大小。比如line-height：2，就表示行高是字号的两倍大；
行高如果小于1，则字体的显示会超出元素的高度。
-->

    # 行高的继承

    如果元素行高单位是纯数字（与字号的倍数关系），那么子元素继承的行高也将是这种与自身字号的倍数关系；
如果元素行高的单位是px，em，百分数，那么子元素继承的行高就不会是这种和子元素字号的倍数关系，而会是元素的具体的计算后的行高数值。

    # 外边距

    外边距属性值可以为正也可以为负。设置负值会导致和其他内容重叠。

    # 外边距折叠

    如果你有两个外边距相接的元素，这些外边距将合并为一个外边距，即最大的单个外边距的大小。

    # margin: 0 auto

    如果手动设置width，但将外边距设为auto，那么这个外边距将进行伸缩以弥补父元素宽度中还未占据的部分。
如果左右外边距都设为auto，那么两个外边距就将设为相等的最大值，这样会导致元素居中。

    flex布局下的margin
父元素display: flex; 里面的子元素margin: auto;在水平竖直两个方向都会居中。（前提是没有其他兄弟元素）
原因和margin: 0 auto一样，四个边距都伸缩以弥补还未占据的部分。

    # 内边距

    不能有负数量的内边距，值必须是0或正的值。

    应用于元素的任何背景都将显示在内边距后面（内边距区域会被背景属性影响），内边距通常用于将内容推离边框。

    # 内/外边距的使用

    父子之间的距离常用父亲的padding。

    兄弟之间的距离常用各自的margin。

    # 边框

    可以通过单独设置四条边框各自属性，设置出有趣图形。

/*直角三角形*/
div{
	width: 0;
	height: 0;
	border: 50px solid transparent;
	border-top-color: red;
	border-bottom: none;
}

/*等边三角形*/
div{
	width: 0;
	height: 0;
	border-top: 30px solid red;
	border-left: 20px solid transparent;
	border-right: 20px solid transparent;
}

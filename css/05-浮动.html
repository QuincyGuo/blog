    标准文档流

    # 标准文档流

    浏览器默认放置元素的系统：

    块级元素，垂直组织，各占据一行，被margin间隔；

    内联元素，只要父块级元素宽度还有空间，就会与其他内联元素、文本内容在同一行，如果空间不够，溢出的元素会移到下一行。

    # 浮动的性质

    # 性质1：浮动的元素脱标

    脱标，就是脱离标准流。

    一旦元素浮动了，不再区分行内、块级元素，将能够并排、设置宽高。

    浮动的元素完全不影响父元素的高度，因为已经不属于文档流的一部分。

    # 性质2：浮动的元素相互贴靠

    float：left的元素，会根据浏览器空间，自动调整向左贴靠。

    如果给一个父容器里面添加两个浮动，两个浮动的宽度超过了父容器的宽度，那么第二个浮动就会从第一个下面另起一行排列。

    # 性质3：收缩

    一个浮动的元素，如果没有设置width，那么将自动收缩为内容的宽度（这点非常像行内元素）。

    # 浮动位置排列梳理

    浮动元素前有标准流元素：

    向上贴靠至前一块级元素下方；若父元素宽度空间足够，将内联元素赶至浮动右边，若不够则向上贴靠至前一内联元素下方；

    浮动元素前面是浮动元素：

    并排排在前一浮动元素右边；若并排宽度超过父元素宽度，则另起一行且向上贴靠上一行里最高的浮动下边框；

    浮动元素后面的标准流元素：

    盒子区域排在前面标准流（包括块级、内联）下方，会与浮动元素的区域重叠，互不干涉，各按各的图层排；但是盒子里的文字及内联元素若父元素宽度足够，会排在前一浮动元素右边。

    <!DOCTYPE html>
<html lang="en">
<head>
    <style>
        .page{
            background-color: tomato;
            width: 200px;
            float: left;
        }
        .test{
            background-color: beige;
            width: 200px;
            height: 70px;
            float: left;
        }
       
       </style>
       <body>
        <p>6666666666</p>
        
        <div class="page">
          <p>6666666666</p>
        </div>
        
        <div class="test">
            <p>6666666666</p>
        </div>
        <div class="test">
          <p>6666666666</p>
      </div>
        <p>6666666666</p>
        
       </body>
</html>

    # 清除浮动

    浮动会造成父元素高度塌陷、与其他浮动、非浮动元素位置改变的问题。

    # 方法1：clear: both

    clear：both，不允许本元素的左侧及右侧有浮动对象。给盒子增加该属性，两侧就不会有浮动对象。

这个clear影响的是元素自己本身，看到两侧有浮动元素他自己会调整位置。
应该将clear属性添加到不希望环绕浮动对象的元素上。因此如果要让一个元素右侧没有浮动元素之后才显示，就为他添加clear：right属性。

    # 方法2：给浮动元素的父元素盒子添加一个高度

    只要浮动元素在一个有高度的盒子内，那这个浮动就不会影响前后的元素（包括浮动与非浮动元素）。
但是，实际开发也不会给每个浮动元素添加盒子父元素，麻烦且不利于布局调整。

    # 方法3：clearfix类库

    给浮动元素的父元素添加 class = “clearfix”;
就可以父元素两侧不会有其他元素显示，父元素也会有高度，等于里面较高的元素高度。

.clearfix::before,
.clearfix::after {
    content: " ";
    display: table;
}
.clearfix::after {
    clear: both;
}
.clearfix {
    *zoom: 1;
}

该方法，只是清除父元素高度塌陷的问题，不清除与其他元素的位置变化问题。。

    # 浮动中的margin

    # margin 不重叠

    标准文档流中，竖直方向的margin不叠加，取较大的值作为margin(水平方向的margin是可以叠加的，即水平方向没有塌陷现象)。
但是如果两个盒子都浮动了，那么两个盒子之间是没有塌陷现象的（水平、竖直方向都没有）。

    # 盒子居中：margin: 0 auto 不能用

    在外边距部分里，盒子设置宽度后，再设置上下margin为0，左右margin为auto，默认会设置为尽可能的大，于是盒子就居中。
只有标准流的盒子才能这样使用，如果盒子浮动了，或者定位了，就没用。

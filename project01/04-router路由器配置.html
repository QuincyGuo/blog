    # NavigationDuplicated错误

    编程式路由跳转，在参数不变情况下，多次进行跳转，会抛出异常，NavigationDuplicated错误。

    重写push、replace方法

    // 将原有的push方法地址，保存起来；
const originPush = VueRouter.prototype.push;
const originReplace = VueRouter.prototype.replace;

// 修改原型的push，让原型的push指向另外一个函数
VueRouter.prototype.push = function (location, onResolved, onRejected) {
  // location就是我们调用 this.$router.push，传递过来的对象
  if (onResolved === undefined && onRejected === undefined) {
    // 证明程序员调用push方法的时候只传递了个匹配路由对象，没有传递成功或者失败的回调，那么抛出的异常没有回调来响应，也就不会出现该错误
    return originPush.call(this, location).catch(() => {});
  } else {
    return originPush.call(this, location, onResolved, onRejected);
  }
};
VueRouter.prototype.replace = function (location, onResolved, onRejected) {
  if (onResolved === undefined && onRejected === undefined) {
    return originReplace.call(this, location).catch(() => {});
  } else {
    return originReplace.call(this, location, onResolved, onRejected);
  }
};

    # 登录后可能跳至不同的页面

    如果去往某些路由组件前需要先登录，则拦截，先强制跳转至登录路由，并在其跳转路径上加上登录成功后要跳往的重定向路径参数； 然后在登录组件的请求跳转那里，进行判断；如果有重定向参数，则跳往重定向参数指定的路由；无参数则正常跳往首页。

    1、全局前置守卫
router.beforeEach((to, from, next) => {
	if (to.path.indexOf("trade") != -1 || to.path.indexOf("pay") != -1 || to.path.indexOf("center") != -1 ) {
      next("/login?redirect=" + toPath);
    }
}
2、登录组件
methods: {
    // 这里是用户登录发请求然后跳转的方法
    async userLogin () {
      try {
        const { phone, password } = this;
        await this.$store.dispatch("userLogin", { phone, password });
        // 服务器返回的result正常，看路由当中是否包含query参数（router里的导航守卫会根据条件添加query参数，使登录之后不往首页跳转）；有参数：跳到query参数指定路由，没有：跳到home
        let toPath = this.$route.query.redirect || '/home';
        this.$router.push(toPath);
      } catch (error) {
        alert(error.message);
      }
    },
},

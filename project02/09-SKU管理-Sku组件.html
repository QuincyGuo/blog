    # 静态组件

    # Layout 布局

    <div>
    <el-row :gutter="20">
        <el-col :span="5">名称</el-col>
        <el-col :span="16">{{skuInfo.skuName}}</el-col>
    </el-row>
    <el-row>
        <el-col :span="5">平台属性</el-col>
        <el-col :span="16">
            <el-tag type="success" style="margin-right:10px"
                    v-for="attr in skuInfo.skuAttrValueList"
                    :key="attr.id"> {{attr.attrName}}-{{attr.valueName}} </el-tag>
        </el-col>
    </el-row>
</div>
// el-row是一行，横向占据父空间的一行，高度由里面的子元素决定；gutter是各列的间隔
// el-col是行里面的列，:span属性是表明该列占据所在行的比例，24为总数，占据x/24；
// el-col里是每列里面的具体元素内容。

    # el-tabs 标签页

    <el-tabs v-model="activeName" @tab-click="handleClick" class="tab">
    <el-tab-pane name="first">
    	<span slot="label" class="fontClass">
    		<i class="el-icon-date">销售额
    	</span>
    </el-tab-pane>
    <el-tab-pane name="second" label="访问量">访问量</el-tab-pane>
</el-tabs>

data () {
	return {
      	activeName: 'first',		// 标题tab的选中名称
    }
},
handleClick (tab, event) {
...
},
// data里需定义一个activeName，其数值与tab-pane的name属性对应，还要有一个初始值。
// handleClick是切换标签页时的触发方法，tab参数是一个vue实例，其上有属性label、name等标识切换后的标签页属性。

// 标签页有两种写法：
// 写法一：tab-pane里定义一个span，就可以自定义标签页名称（给其加图标、图片等）
// 写法二：正常只用tab-pane定义名称。

    # 进入离开组件

    # 数据

    名称 数据类型 所属 数据介绍
    skuInfo Object data 待查看的sku实例数据
    allSkuList Array data 全部的SKU实例数据
    currentPage number data spu展示列表-当前页
    pageDataCountNow number data spu展示列表-每页展示的数据个数
    total number data spu展示列表-所有的数据个数
    drawer Boolean data 抽屉组件的显示/隐藏

    # 生命周期

    1、mounted ()

    调用getSkuList方法；组件向服务器发送”获取全部sku实例“的网络请求，如果服务器返回的result正常，则将返回的数据保存到组件的data中，以供后续使用。

    # API

    1、获取某一页sku列表

    // 路径：/admin/product/list/{page}/{limit}	请求方式：get	参数：page 当前页码	limit 每页数据个数
export const reqAllSkuList = (page, limit) =>
  request({ url: `/admin/product/list/${page}/${limit}`, method: "get" });

    2、上架sku

    // 路径：/admin/product/onsale/{skuId}		请求方式：get	参数：skuId
export const reqSale = (skuId) =>
  request({ url: `/admin/product/onSale/${skuId}`, method: "get" });

    3、下架sku

    // 路径：/admin/product/cancelsale/{skuId}		请求方式：get	参数：skuId
export const reqCancelSale = (skuId) =>
  request({ url: `/admin/product/cancelSale/${skuId}`, method: "get" });

    4、通过skuId获取sku详情数据

    // 路径：/admin/product/getSkuById/{skuId}		请求方式：get	参数：skuId
export const reqGetSkuById = (skuId) =>
  request({ url: `/admin/product/getSkuById/${skuId}`, method: "get" });

    # methods

    1、getSkuList () 获取某一页sku实例数据

    1、参数：无

2、函数内容： 
2.1、调用API-获取某一页SKU实例，等待服务器返回结果；如果返回结果成功，将返回的数据保存到data中；
2.2、失败则提示失败原因。

    2、getSkuInfo (row) 查看sku实例详情

    1、参数：row 待查看的sku实例相关数据

2、函数内容： 
2.1、调用API-通过skuId获取sku详情数据，等待服务器返回结果；如果返回结果成功，将返回的数据保存到data中；
2.2、显示drawer组件，在其上面显示sku实例详情数据。

    3、sale (row) 将sku实例上架

    1、参数：row 待上架的sku实例相关数据

2、函数内容： 
2.1、调用API-上架sku，等待服务器返回结果；如果返回结果成功，将返回的数据保存到data中；
2.2、弹出消息框提示上架成功。

    4、cancelSale(row) 将sku实例下架

    1、参数：row 待下架的sku实例相关数据

2、函数内容： 
2.1、调用API-下架sku，等待服务器返回结果；如果返回结果成功，将返回的数据保存到data中；
2.2、弹出消息框提示下架成功。

    5、handlerCurrentChange (page) 切换页码

    1、参数：page 切换后新的页码

2、函数内容：
2.1、将切换页码后新的页码page赋值给data:currentPage；
2.2、调用methods:getSkuList()，获取本页的sku列表数据

    6、handleSizeChange(count) 切换每页列表数据的个数

    1、参数：count 切换后新的每页数据的个数

2、函数内容：
2.1、将切换页码后新的每页数据的个数count赋值给data:pageDataCountNow；
2.2、调用methods:getSkuList()，获取本页的sku列表数据

    # 功能点

    1、页面数据初始化

    组件在mounted阶段，调用页面数据初始化函数，函数内调用获取sku列表数据的API，向服务器发送请求；服务器返回成功则将返回的数据保存到组件data中，页面显示时使用相应数据。

    用户使用过程中，切换页码、切换每页数据个数等操作，组件获取到当前页码等数据；调用获取sku列表数据的API...（与上面一致）。

    2、sku上架/下架

    点击某条sku实例的上架/下架按钮，调用sku上/下架API，向服务器发送请求；服务器返回成功则改变data中的上/下架状态数据。

    3、查看某条sku实例的详情数据

    用户点击查看按钮，调用获取sku实例详情数据的API，向服务器发送请求；服务器返回成功则将返回的数据保存到组件data中；将drawer组件切换为显示状态，页面显示时使用相应数据。
